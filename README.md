# LYRA — Conversation Viewer

PWA для просмотра и управления JSON-экспортами из VRS/A (Artist songwriting studio).

---

## Структура проекта

```
LYRA-APP/
├── index.html              # точка входа
├── manifest.json           # PWA манифест
├── sw.js                   # Service Worker (офлайн-кэш)
├── favicon.ico
├── assets/
│   ├── css/
│   │   └── main.css        # все стили
│   ├── js/
│   │   ├── app.js          # точка входа JS, файловый ввод, поиск
│   │   ├── parser.js       # парсинг user/assistant сообщений
│   │   ├── render.js       # построение карточек запросов
│   │   ├── sidebar.js      # навигация, библиотеки, drag & drop
│   │   └── storage.js      # база данных (localStorage)
│   └── icons/
└── README.md
```

## Функционал

### Загрузка файлов

- Кнопка **↑ Загрузить** в хедере
- Drag & drop JSON-файла на экран загрузки
- Поддерживается формат экспорта VRS/A: `{ messages: [...], exportedAt: "..." }`
- После загрузки файл автоматически сохраняется в базе данных (localStorage)

### Именование файлов

- После загрузки файл автоматически получает имя в формате **DD.MM.YYYY** на основе поля `exportedAt` из JSON (или текущей даты, если поле отсутствует)
- Если файл с такой датой уже существует — добавляется суффикс: `03.05.2025 (2)`
- **Двойной клик** по названию файла в сайдбаре → редактируемое поле → Enter или клик вне поля сохраняет новое имя

### База данных файлов

- Все загруженные файлы хранятся в `localStorage` под ключом `lyra_files_v3`
- При следующем открытии приложения последний просмотренный файл восстанавливается автоматически
- Каждый файл хранит: id, имя, дату сохранения, количество запросов, id библиотеки

### Библиотеки (папки)

- Кнопка **+ библиотека** в сайдбаре создаёт новую папку
- Библиотека раскрывается/сворачивается кликом на заголовок
- **Двойной клик** по названию библиотеки → переименование
- Кнопка **✕** удаляет библиотеку (файлы остаются, перемещаются в "Без библиотеки")
- Рядом с названием библиотеки — счётчик файлов внутри

#### Перемещение файлов в библиотеку

Два способа:

1. **Drag & drop** — перетащи файл за иконку `⠿` в нужную библиотеку или в "Без библиотеки"
2. **Кнопка ↗** — появляется при наведении на файл → выпадающее меню со списком библиотек

### Просмотр запросов

После открытия файла в основной области отображаются пары запрос/ответ.

#### Режим отображения по умолчанию

Каждая пара показывает только **Lyrics** (ответ ассистента). Остальное свёрнуто:

- **▸ Production** — кнопка раскрывает блок с production-описанием трека
- **↓ показать запрос** — кнопка раскрывает USER-карточку со всеми параметрами

#### Карточка USER содержит

- **Lyrics Settings**: Artist Name, Core Theme, Mood Tag, Banned Words, Length, Explicit Language
- **Rhyme Controls**: Rhyme Density, Rhyme Complexity, Rhyme Placement, Rhyme Quality, Structure Patterns, Poetic Forms
- **Prompt**: полный текст запроса

#### Карточка Assistant содержит

- **Model Info**: модель, temperature, top P (в хедере карточки)
- **Production**: описание битa / жанра / инструментов (свёрнуто)
- **Lyrics**: текст трека (всегда виден)

### Копирование

- Кнопка **copy** на каждом поле — копирует содержимое блока в буфер обмена
- Кнопка **⎘ скопировать всё** в заголовке пары — копирует Production + Lyrics одним текстом с метками `--- PRODUCTION ---` и `--- LYRICS ---`

### Поиск

- Sticky-строка поиска появляется после загрузки файла
- Ищет по: Artist Name, Core Theme, первым 300 символам Lyrics
- Debounce 180ms — не перегружает интерфейс при быстром вводе
- **Escape** очищает поиск
- Показывает счётчик найденных пар

### Навигация в сайдбаре

- После открытия файла под ним раскрывается список запросов `#1 Eminem`, `#2 Eminem`...
- Клик по пункту — плавный скролл к нужной паре
- Активный пункт подсвечивается при скролле (IntersectionObserver)
- Если в файле несколько разных артистов — появляются чипы-фильтры: `все / Artist1 / Artist2`

### Удаление

- Кнопка **✕** на файле (появляется при наведении) — удаляет файл с подтверждением
- Кнопка **✕** на библиотеке — удаляет только библиотеку, файлы перемещаются в "Без библиотеки"

## Хранилище (localStorage)

| Ключ | Описание |
|------|----------|
| `lyra_files_v3` | Массив файлов: id, name, data (JSON-строка), savedAt, pairCount, libId |
| `lyra_libs_v1` | Массив библиотек: id, name, createdAt |

> Внимание: localStorage ограничен ~5MB на домен. При больших файлах возможны ошибки сохранения — в консоли появится предупреждение `LYRA storage: ...`

## PWA

- Устанавливается как приложение на Android (Chrome) и iOS (Safari → Добавить на экран)
- Service Worker кэширует все статичные ресурсы для офлайн-доступа
- `start_url` и `scope` настроены на `/LYRA-APP/`

## Деплой на GitHub Pages

1. Загрузи все файлы в репозиторий `LYRA-APP`
2. Settings → Pages → Source: `main` branch, `/ (root)`
3. Приложение доступно на `https://kartg33.github.io/LYRA-APP/`

> После обновления файлов браузер может показывать кэшированную версию. Хард-рефреш: **Ctrl+Shift+R** или открой DevTools → Application → Service Workers → Unregister, затем перезагрузи.

## Формат JSON-экспорта (VRS/A)

```json
{
  "exportedAt": "2025-05-03T19:00:00.000Z",
  "messages": [
    {
      "role": "user",
      "content": "artist_name: Eminem, core_theme: ..., rhyme_schemes: ..."
    },
    {
      "role": "assistant",
      "content": "[Model: claude-opus-4 | Temperature: 0.9 | Top P: 0.9]\n[Intro | ...]\n..."
    }
  ]
}
```

Парсер (`parser.js`) автоматически извлекает параметры из строки пользователя и метаданные модели из строки ассистента.
